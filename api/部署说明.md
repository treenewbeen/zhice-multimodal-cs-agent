# 线上服务部署说明

本服务（`api/app.py`）复用源码包核心智能体，**需保持整个源码包目录结构一起部署**
（`api/` 依赖同级 `../src` 与 `../knowledge_base`，路径自动定位，无需改代码）。

---

## 一、环境准备

- 服务器：Linux（推荐 Ubuntu 20.04+），Python 3.10+，建议 ≥ 4GB 内存（知识库与插图常驻）。**无需 GPU**（检索为纯 Python，生成走外部网关）。学生/试用档的 2核4G 轻量服务器即可。
- 上传整个 **`知册_客服智能体_源码包/`** 到服务器（含 `src/`、`knowledge_base/`、`api/`，**勿拆目录**；路径以包根为基准自动定位）。
- **在云厂商安全组 / 防火墙放行 TCP 8000 端口**（最易漏的一步！否则即使有公网 IP 也连不上服务）。

```bash
cd 知册_客服智能体_源码包
python3 -m venv .venv && source .venv/bin/activate
pip install -r api/requirements.txt
```

## 二、配置密钥（在包根目录创建 `.env`）

```bash
cp .env.example .env     # 然后编辑 .env，填入 TEXT_*/EMBEDDING_*/JUDGE_* 等密钥
# 生产部署建议设置鉴权令牌，/chat 将校验 Authorization: Bearer
echo 'KAFU_API_TOKEN=<KAFU_API_TOKEN>' >> .env
```

> `.env` 由 `runtime.load_env()` 读取（在包根目录）。**源码提交不含 `.env`/密钥**；部署时单独配置。

## 三、启动服务

**启动前自检（确认密钥可用 + 服务器能出网调网关）：**
```bash
# 在包根目录执行；能正常打印答案 = .env 密钥有效且服务器可出网
python src/agent.py one "如何给空调遥控器安装电池？"
```
> ⚠️ 高频调用全程靠 `.env` 里的网关按 token 计费——上线前务必确认密钥**有余额**。

**开发 / 快速验证：**
```bash
cd api
uvicorn app:app --host 0.0.0.0 --port 8000
```

**生产（多进程，推荐）：**
```bash
cd api
gunicorn -k uvicorn.workers.UvicornWorker -w 2 -b 0.0.0.0:8000 app:app
```

**后台常驻（systemd / nohup 任选）：**
```bash
cd api && nohup gunicorn -k uvicorn.workers.UvicornWorker -w 2 -b 0.0.0.0:8000 app:app > ../cache/api.log 2>&1 &
```

## 四、验证

```bash
curl http://<服务器IP>:8000/health
curl -X POST http://<服务器IP>:8000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <KAFU_API_TOKEN>" \
  -d '{"question":"如何给空调遥控器安装电池？"}'
```
- 交互式文档：浏览器打开 `http://<服务器IP>:8000/docs`（Swagger UI，可直接点测）。
- 图片在线查看：响应 `data.image_urls` 即可直接访问。

## 五、Docker 部署（可选，见同目录 `Dockerfile`）

```bash
# 在源码包根目录执行
docker build -f api/Dockerfile -t zhice-cs-agent .
# ⚠️ 用 -v 挂载 .env，不要用 --env-file：.dockerignore 已排除 .env，
#    且 runtime.load_env() 只读包内 .env 文件、不读系统环境变量，
#    --env-file 注入的变量喂不进模型密钥，会导致容器里 LLM 调用失败、答不出来。
docker run -d -p 8000:8000 -v "$(pwd)/.env:/app/.env" --name zhice zhice-cs-agent
```

## 六、反向代理 / HTTPS（可选）

生产建议用 Nginx 反代到 `127.0.0.1:8000` 并配置 HTTPS（Let's Encrypt）。`/chat` 超时建议 30s（多模态）。

---

### 常见问题
- **首次请求慢**：启动时预载知识库 + 首次调用外部 LLM；之后命中 `cache/` 磁盘缓存。
- **无图片返回**：确认 `knowledge_base/images/` 已随包上传（image_id→URL 依赖该目录）。
- **401**：请求未带正确 `Authorization: Bearer <KAFU_API_TOKEN>`。
- **LLM 调用失败**：检查 `.env` 网关地址与密钥、服务器出网。
