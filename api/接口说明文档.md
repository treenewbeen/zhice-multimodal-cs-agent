# 知册 · 多模态客服智能体 — API 接口说明文档

> 线上部署服务，供在线访问与集成调用。核心端点：`POST /chat`（多模态对话交互）。
> 接口定义如下。

## 1. 基础信息

| 配置项 | 值 |
|---|---|
| 接口类型 | RESTful API（无状态，适配高并发/分布式） |
| 核心端点 | `POST /chat` |
| 辅助端点 | `POST /verify`（接地校验）、`GET /health`（健康检查）、`GET /images/{文件名}`（插图在线查看） |
| 请求方式 | POST（`/chat`、`/verify`） |
| 通信协议 | HTTP/1.1（测试）、HTTPS（生产） |
| 字符编码 | UTF-8 |
| 认证方式 | Bearer Token（生产部署建议设置；由 `KAFU_API_TOKEN` 配置） |

## 2. 认证规范

服务端通过包根目录 `.env` 或进程环境变量配置 `KAFU_API_TOKEN`。生产部署时，所有 `/chat` 请求必须在头部携带：

```http
Authorization: Bearer {KAFU_API_TOKEN}
```

本地临时调试可以留空 `KAFU_API_TOKEN`，但生产部署建议务必设置。

## 3. 核心端点 `POST /chat`

### 3.1 请求头（Request Header）

| 字段 | 必选 | 类型 | 说明 | 示例 |
|---|---|---|---|---|
| Content-Type | 是 | String | 固定 JSON | `application/json` |
| Authorization | 是 | String | Bearer Token | `Bearer sk_xxx` |
| X-Request-Id | 否 | String | 请求唯一标识，用于追溯 | `kf_req_123e4567` |
| X-Client-Type | 否 | String | 调用方终端标识 | `web` / `app/ios` |

### 3.2 请求体（Request Body）

| 字段 | 必选 | 类型 | 默认 | 说明 |
|---|---|---|---|---|
| question | 是 | String | - | 用户问题字符串（长度 > 1） |
| images | 否 | Array\<String\> | `[]` | Base64 图片列表，支持 0-3 张，每张 ≤ 5MB；必须带前缀 `data:image/{png/jpg/jpeg/webp};base64,{编码}` |
| session_id | 否 | String | 自动生成 | 会话 ID，用于多轮对话续接 |
| stream | 否 | Boolean | `false` | 是否流式（本实现同步返回完整答案） |

**示例 1 · 极简文本**
```json
{ "question": "如何给空调遥控器安装电池？" }
```

**示例 2 · 多模态（文本 + 图片）**
```json
{
  "question": "我的设备屏幕显示这个，是什么意思？",
  "images": ["data:image/png;base64,iVBORw0KGgo..."],
  "session_id": "kf_session_889900"
}
```

**示例 3 · 多轮续接**
```json
{ "question": "那它怎么清洁？", "session_id": "kf_session_889900" }
```

### 3.3 响应体（成功，标准信封）

```jsonc
{
  "code": 0,
  "msg": "success",
  "data": {
    "answer": "安装电池 使用遥控器前，请先安装电池… <PIC> …",  // 答案正文，<PIC> 标记图片插入位置
    "image_ids": ["Manual01_5"],                                 // 图片 id，与 <PIC> 顺序一一对应
    "image_urls": ["http://<host>/images/Manual01_5.png"],       // 图片在线 URL，可直接访问查看
    "session_id": "kf_session_889900",
    "timestamp": 1741008000,
    "meta": {                                                    // 可解释溯源：便于追溯
      "type": "manual", "product_key": "zh_空调",
      "conf": "high", "policy": "compose_high", "sec": 12
    },
    "grounding": { "grounded_rate": 1.0, "ungrounded": [] }       // 事后接地校验（抗幻觉）
  }
}
```

> 说明：`answer` 中的 `<PIC>` 占位符按顺序对应 `image_ids` / `image_urls`，客户端可在该位置渲染对应图片。

### 3.4 错误响应

| HTTP 状态 | 含义 |
|---|---|
| 400 | `question` 为空 / 长度不足，或 `images` 超过 3 张 |
| 401 | Bearer Token 缺失或错误（仅在设置了 `KAFU_API_TOKEN` 时） |
| 500 | 服务内部错误 |

## 4. 辅助端点

- `POST /verify`　body `{ "answer": "...", "evidence": "..." }` → 返回 `{ grounded_rate, ungrounded[] }`（事后接地校验演示）。
- `GET /health` → `{ "ok": true, ... }`。
- `GET /images/{文件名}` → 在线查看知识库插图。

## 5. cURL 调用示例

```bash
# 文本对话
curl -X POST http://<host>:8000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <KAFU_API_TOKEN>" \
  -d '{"question":"如何给空调遥控器安装电池？"}'

# 多模态对话
curl -X POST http://<host>:8000/chat \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <KAFU_API_TOKEN>" \
  -d '{"question":"我的设备屏幕显示这个，是什么意思？","images":["data:image/png;base64,iVBORw0KGgo..."]}'
```

> 在线交互式文档：服务启动后访问 `http://<host>:8000/docs`（FastAPI 自动生成 OpenAPI/Swagger UI）。
